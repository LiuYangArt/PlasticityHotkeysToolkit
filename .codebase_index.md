# Codebase Index

## 仓库概览

这是一个轻量工具仓库，用来给已安装的 Plasticity 注入自定义快捷键。

仓库分为 4 个层次：

1. 顶层入口
2. `tools/` 启动与安装脚本
3. `hotkeys/` 注入与运行时代码
4. `docs/` 研究、方案、排障与使用文档

没有构建系统、没有测试框架、没有依赖安装步骤。  
主要验证方式是语法检查、JSON 检查和真实运行验证。

## hotkeys/

### `hotkeys/custom-shortcuts.json`

- 当前唯一快捷键配置源
- 按 selector 分组配置 chord -> commandId
- 目前包含：
  - `enter` -> `custom:outliner:activate-selected`
  - `alt-shift-4` -> `custom:outliner:activate-parent-folder`
  - `ctrl-alt-shift-x` -> `custom:outliner:delete-empty-groups`
  - `shift-b` -> `custom:transform:set-pivot-bbox`

### `hotkeys/inject-main.mjs`

- 角色：main process 注入器
- 做的事：
  - 连接 Electron inspector
  - 读取 JSON 配置和 renderer 脚本
  - 拼装 payload
  - 把 payload 注入所有 BrowserWindow
  - 等待 renderer 状态返回
- 什么时候改：
  - 注入协议变化
  - 调试输出变化
  - 启动/安装脚本参数变化

### `hotkeys/renderer-hotkeys.js`

- 角色：真正的运行时代码核心
- 做的事：
  - 安装 `keydown` / `pointerdown` 监听
  - 维护 debug overlay / toast / status
  - 解析 chord
  - 做 selector 命中
  - 执行自定义命令
  - 基于 DOM 结构做点击、双击、菜单命中
- 重要注意：
  - 改这个文件时必须同步提升 `VERSION`
  - 这是最容易引入回归的文件
- 当前重要命令：
  - `custom:outliner:activate-selected`
  - `custom:outliner:activate-parent-folder`
  - `custom:outliner:delete-empty-groups`
  - `custom:transform:set-pivot-bbox`

## tools/

### `tools/launch-plasticity-hotkeys.ps1`

- 角色：统一启动入口
- 做的事：
  - 区分 `beta` / `stable`
  - 定位 Plasticity 安装根目录
  - 识别真实可执行文件路径
  - 决定是否复用已有 `--inspect` 进程
  - 调用 `hotkeys/inject-main.mjs`
- 这是最关键的 PowerShell 脚本

### `tools/install-hotkeys-toolkit.ps1`

- 角色：把当前仓库同步安装到 `%LOCALAPPDATA%\\PlasticityHotkeysToolkit`
- 使用场景：
  - 重装系统后快速恢复
  - 从别的目录开发后重新发布到固定目录

### `tools/debug-hotkeys-status.ps1`

- 角色：读取当前注入状态
- 用途：
  - 看 renderer 版本
  - 看当前配置是否生效
  - 看 debug 模式是否开启

### 其他 `.cmd` 包装脚本

- 顶层日常入口：
  - `start-plasticity-beta.cmd`
  - `start-plasticity-stable.cmd`
- 调试入口：
  - `tools/start-plasticity-beta-debug.cmd`
  - `tools/start-plasticity-stable-debug.cmd`
- 状态入口：
  - `tools/debug-hotkeys-status-beta.cmd`
  - `tools/debug-hotkeys-status-stable.cmd`
- 安装入口：
  - `tools/install-hotkeys-toolkit.cmd`

这些 `.cmd` 都很薄，真正逻辑基本都在 `.ps1` 里。

## docs/

### `docs/how-to-add-hotkeys.md`

- 角色：新增快捷键的操作说明
- 优先在新增命令前先看这份

### `docs/plasticity-shortcut-system-research.md`

- 角色：前期研究记录
- 适合在需要重新理解整体注入方案时看

### `docs/plasticity-unofficial-shortcut-plan.md`

- 角色：非官方快捷键扩展计划
- 适合在需要理解整体实施路线时看

### `docs/postmortem-2026-04-08-outliner-delete-empty-groups.md`

- 角色：一次真实问题的复盘
- 重点经验：
  - renderer 版本守卫
  - 不要依赖脆弱 DOM 假设
  - 先区分输入链路 / 命令分发 / DOM 命中

### `docs/postmortem-2026-04-08-move-pivot-bbox.md`

- 角色：一次 `Move > Pivot > Bbox` 快捷键实现复盘
- 重点经验：
  - 工具参数面板优先按根节点和分组标题定位
  - tooltip 文本比图标和绝对位置更稳定
  - renderer 运行时代码有改动时仍然必须同步提升 `VERSION`

## 高风险点

- 忘记提升 `renderer-hotkeys.js` 里的 `VERSION`
- 在没有 debug 验证的情况下修改 DOM 命中逻辑
- 把顶层入口脚本和 `tools/` 内部脚本的路径关系改坏
- 重新引入多份快捷键 JSON，造成配置源分裂
